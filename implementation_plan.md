# 구현 계획 - 인투처치 (Into Church) 리디자인

이 프로젝트의 목표는 "투명성, 모던한 미학, 인터랙티브 요소"에 초점을 맞춰, 30대 타겟의 스타트업 분위기가 나는 웹사이트로 현대화하는 것입니다.

## 제안된 변경 사항

### 기술 스택 및 설정
#### [신규] 프론트엔드 빌드 파이프라인
- **Node.js**: 프로젝트 루트에 `package.json` 초기화.
- **Tailwind CSS**: CDN 방식에서 로컬 빌드 방식으로 변경.
    - `tailwindcss`, `postcss`, `autoprefixer` 설치.
    - `ministry/templates/**/*.html`을 바라보도록 `tailwind.config.js` 설정.
    - `static/src/input.css` -> `static/css/output.css` 빌드 구성.
- **JavaScript 라이브러리**:
    - **Alpine.js**: 경량 인터랙션 (마이크로 연출, 모달 토글).
    - **Chart.js**: "투명성" 데이터 시각화.
    - **AOS (Animate On Scroll)**: 스크롤 애니메이션.

### 컴포넌트 아키텍처
#### [수정] 글로벌 레이아웃 (`base.html`)
- 생성된 `output.css` 통합.
- 네비게이션 업데이트 (Sticky, 투명 -> 불투명 전환).
- 소셜 링크가 포함된 확장형 푸터(Footer) 추가.
- "다크 모드" 토글 상태 처리 (Alpine.js store 활용).

#### [신규] 홈페이지 구현 (`index.html` 또는 `home.html`)
- **히어로 섹션**:
    - 비디오 배경 또는 고품질 그라데이션.
    - 애니메이션 타이포그래피 (헤드라인: "믿음과 투명성이 만나는 곳").
    - 행동 유도(CTA) 버튼.
- **스토리 섹션**:
    - 타임라인/인포그래픽 스타일의 "투명성" 내러티브.
- **데이터 대시보드**:
    - 출석 및 헌금 통계를 위한 Chart.js 통합.
- **참여 모듈**:
    - "라이브" 상태 표시기.
    - 리뷰/후기 그리드 카드 레이아웃.
    - 소셜 프루프 카운터.

### 핵심 기능
- **실시간 요소**: "Live" 상태에 대한 모의(Mocked) 데이터 또는 간단한 폴링.
- **인터랙티브 폼**: Alpine.js 트랜지션을 활용한 온보딩 모달.

### 기술적 과제
- **반응형 디자인**: 모바일 우선(Mobile-First) Tailwind 클래스 적용.
- **최적화**: CSS를 위한 `django-compressor` 또는 간단한 빌드 단계 사용. 가능한 경우 에셋을 WebP로 변환.

## 검증 계획

### 자동화된 검사
- `npm run build`: 오류 없이 Tailwind 빌드가 완료되는지 확인.
- `python manage.py check`: Django 설정 확인.

### 수동 검증
- **시각적 확인**:
    - 히어로 애니메이션 로딩 확인.
    - 다크/라이트 모드 토글 테스트.
    - 모바일 뷰(375px 너비)에서의 반응형 확인.
- **기능 확인**:
    - 캐러셀 네비게이션 작동 확인.
    - 샘플 데이터를 사용한 차트 렌더링 확인.
